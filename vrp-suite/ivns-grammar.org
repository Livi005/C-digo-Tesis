* Descripción

Este archivo reúne en un solo lugar el código de la gramática usada para IVNS.

Se agrupan cuatro partes:

1. definición del paquete IVNS;
2. generación de cadenas válidas según la gramática;
3. verificación y desambiguación de la cadena generada;
4. traducción de acciones IVNS a criterio ejecutable del VRP.

La idea general es:

- generar una cadena válida;
- verificar si hay ambigüedad estructural;
- convertir la cadena en una lista de acciones simbólicas;
- dejar esa información lista para conectarla con el VNS y el VRP;
- ejecutar ese criterio sobre una solución real del VRP.

* Mapa de símbolos del lenguaje IVNS

La gramática de IVNS genera cadenas formadas por símbolos simples. Cada
símbolo representa una operación básica sobre una solución del CVRP.

El significado usado en este proyecto es:

| Símbolo | Significado conceptual | Acción sobre la solución |
|---------+------------------------+--------------------------|
| =r=     | seleccionar ruta       | elige una ruta del conjunto de rutas |
| =a=     | seleccionar cliente    | elige un cliente de una ruta |
| =b=     | insertar cliente       | inserta un cliente previamente seleccionado |
| =c=     | intercambiar clientes  | intercambia dos clientes seleccionados |
| =f=     | seleccionar subruta    | elige una secuencia de clientes dentro de una ruta |
| =g=     | insertar subruta       | inserta una subruta previamente seleccionada |
| =e=     | intercambiar subrutas  | intercambia dos subrutas seleccionadas |

Por ejemplo, una cadena como =rab= puede interpretarse como:

1. seleccionar una ruta;
2. seleccionar un cliente de esa ruta;
3. insertar ese cliente en una ruta.

Una cadena como =rffe= puede interpretarse como:

1. seleccionar una ruta;
2. seleccionar dos subrutas;
3. intercambiar esas subrutas.

Este mapa permite entender que IVNS no genera movimientos manualmente,
sino secuencias de acciones que después se traducen a criterios de vecindad
del framework VRP.

(defun cadena-balanceada-p
       (cadena
        &key
          (limite-desequilibrio
           (get-config :max-imbalance 2)))

  "Verifica que CADENA no esté dominada excesivamente por un símbolo.

   Esta función se carga temprano porque es usada por los validadores
   de cadenas IVNS. No exige que todos los símbolos aparezcan; solo evita
   cadenas demasiado desbalanceadas en frecuencia."

  (if (or (null cadena)
          (not (stringp cadena))
          (= (length cadena) 0))

      nil

      (let* ((symbols
              '(#\r #\a #\b #\c #\e #\f #\g))

             (counts
              (remove
               0
               (mapcar
                (lambda (s)
                  (count s cadena))
                symbols))))

        (if (null counts)

            nil

            (<= (- (apply #'max counts)
                   (apply #'min counts))
                limite-desequilibrio)))))

* Archivo: ivns-string-generator.lisp

Esta sección contiene la gramática y el generador de cadenas IVNS.

El archivo generado es:

- =../src/ivns-string-generator.lisp=

La responsabilidad de esta parte es producir cadenas candidatas que
representen posibles criterios de vecindad.

La gramática define qué combinaciones de símbolos están permitidas. Sin
embargo, no toda cadena generada es conveniente para ejecutar un
experimento. Algunas cadenas pueden ser demasiado largas, repetitivas,
costosas o poco útiles.

Por eso esta sección incluye también filtros para controlar:

- longitud máxima;
- cantidad máxima de rutas seleccionadas;
- balance entre símbolos de selección e inserción;
- complejidad estimada;
- patrones que suelen producir vecindarios demasiado costosos;
- duplicados.

La salida esperada de esta sección es una cadena IVNS válida, simple y
ejecutable.

** Variables principales

- =p-productions=: gramática completa; define qué puede producir cada símbolo no terminal.
- =terminals=: terminales del lenguaje; son los símbolos finales que aparecerán en la cadena.
- =non-terminals=: símbolos auxiliares que se expanden usando la gramática.
- =generated-strings=: historial de cadenas ya producidas; se usa para evitar repetir resultados.
- =current-index=: contador auxiliar para anotar terminales indexados.
- =*r-limit*= límite del número de =r= permitidas en una cadena.
- =*r-count*= contador temporal de cuántas =r= han salido en el intento actual.

** Qué representa la gramática

La variable =p-productions= es el núcleo formal del módulo. 
En ella se define el lenguaje de criterios que el IVNS puede generar. 
Cada producción indica qué combinaciones de símbolos están permitidas y, 
por tanto, qué patrones de acciones podrán existir después en la interpretación.

** Funciones auxiliares pequeñas

Las funciones pequeñas de esta sección cumplen tareas muy concretas y reutilizables:

- =get-productions=: devuelve las producciones asociadas a un símbolo.
- =terminal-p=: decide si un símbolo ya es terminal y no debe seguir expandiéndose.
- =non-terminal-p=: decide si un símbolo todavía debe expandirse.
- =string-seen-p=: verifica si una cadena ya había sido generada.
- =add-string=: guarda una cadena nueva en el historial.
- =generate-indexed-terminal=: crea una variante indexada de un terminal.
- =choose-random=: selecciona un elemento aleatorio de una lista.
- =flatten-list=: aplana listas anidadas para dejar la salida en forma simple.

Estas funciones existen para que el generador principal se mantenga legible. 
En lugar de concentrar toda la lógica en un único bloque largo, el código separa tareas pequeñas y bien definidas.

       (string)

   Se consulta directamente la variable global GENERATED-STRINGS.
   No se usa un parámetro con el mismo nombre porque GENERATED-STRINGS
   es especial por DEFparameter y una vinculación local puede ocultar

          generated-strings

       (string)

   La cadena se añade solo si no estaba registrada previamente.
   Devuelve la misma cadena para permitir usar esta función en flujos

    (push string
          generated-strings))

(defun ivns-production-efg-score (production)
  "Mide cuánto favorece una producción a las operaciones de subrutas.

   F: selección de subruta
   G: inserción de subruta
   E: intercambio de subrutas

   Se da más peso a E porque suele representar una operación más fuerte."
  (+ (ivns-symbol-count-in-production 'F production)
     (ivns-symbol-count-in-production 'G production)
     (* 2
        (ivns-symbol-count-in-production 'E production))))


(defun ivns-production-weight (symbol production)
  "Calcula el peso de una producción durante la generación.

   Si :prefer-efg-productions está activo, las producciones con F/G/E
   tienen mayor probabilidad de ser seleccionadas.

   Si :prefer-long-productions está activo, las producciones más largas
   también reciben un pequeño refuerzo."
  (let ((weight 1))

    ;; Favorecer producciones con f/g/e.
    (when (and (member symbol '(S S1))
               (get-config :prefer-efg-productions nil))
      (incf weight
            (* (get-config :efg-production-weight 3)
               (ivns-production-efg-score production))))

    ;; Favorecer producciones más largas, pero sin exagerar.
    (when (and (member symbol '(S S1))
               (get-config :prefer-long-productions nil))
      (incf weight
            (max 0
                 (- (length production) 3))))

    ;; Evitar peso cero o negativo.
    (max 1 weight)))


(defun choose-production-for-symbol (symbol productions)
  "Selecciona una producción para SYMBOL.

   Por defecto se comporta como selección aleatoria uniforme.
   Cuando se activan perfiles EFG, usa selección ponderada para favorecer
   cadenas con operaciones de subrutas."
  (if (or (not (get-config :prefer-efg-productions nil))
          (not (member symbol '(S S1))))
      (choose-random productions)

      (let* ((weighted
              (mapcar
               (lambda (production)
                 (cons production
                       (ivns-production-weight symbol production)))
               productions))

             (total
              (reduce #'+
                      weighted
                      :key #'cdr))

             (ticket
              (random total))

             (acc 0))

        (dolist (pair weighted)
          (incf acc
                (cdr pair))

          (when (< ticket acc)
            (return
              (car pair)))))))

(defun flatten-list (nested-list)
  "Convierte una lista anidada en una lista plana de elementos."

               ((null element) result)
               ((listp element)
                (flatten-recursive (first element) (flatten-recursive (rest element) result)))
               ((stringp element) (cons element result))
               (t result))))
    (flatten-recursive nested-list nil)))

*** =generate-from-symbol=

Es la función central del generador. Su papel es expandir recursivamente un símbolo de la gramática hasta obtener terminales.

A grandes rasgos:

1. si recibe un terminal, lo devuelve;
2. si recibe =epsilon=, no produce nada;
3. si recibe un no terminal, elige una producción válida al azar;
4. expande recursivamente cada símbolo de esa producción;
5. concatena el resultado.

Además, dentro de esta expansión controla restricciones especiales, como el número máximo de símbolos =r=.

#+BEGIN_SRC lisp :tangle ../src/ivns-string-generator.lisp

(defun generate-from-symbol
       (symbol
        &key
          (productions p-productions)
          (term-list terminals)
          (non-term-list non-terminals)
          (index current-index))

  "Expande recursivamente SYMBOL en secuencia de terminales según la gramática.

   Cambios importantes:
   - respeta :max-r desde *ivns-config*;
   - puede usar selección ponderada para favorecer f/g/e;
   - mantiene el comportamiento original si no hay perfil EFG activo."

  (cond

    ((terminal-p symbol :terminals term-list)

     ;; Si el terminal es \"r\", contamos y cortamos el intento si excede
     ;; el límite configurable.
     (when (and (stringp symbol)
                (string= symbol "r"))

       (incf *r-count*)

       (when (> *r-count*
                (get-config :max-r *r-limit*))
         (error 'r-limit-exceeded)))

     (if (and (>= (length symbol) 3)
              (string= (subseq symbol
                               (- (length symbol) 2))
                       "_i"))

         (let ((indexed-terminal
                (generate-indexed-terminal
                 symbol
                 :current-index index)))

           (incf index)

           (list
            (list
             (subseq indexed-terminal 0 1)
             index)))

         (list
          (list symbol))))

    ((eq symbol 'epsilon)
     nil)

    ((non-terminal-p symbol :non-terminals non-term-list)

     (let ((prods
            (get-productions
             symbol
             :p-productions productions)))

       (if prods

           (let ((chosen-production
                  (choose-production-for-symbol
                   symbol
                   prods)))

             (flatten-list

              (mapcar
               (lambda (s)
                 (generate-from-symbol
                  s
                  :productions productions
                  :term-list term-list
                  :non-term-list non-term-list
                  :index index))

               chosen-production)))

           (error
            "No hay producción para el símbolo ~a"
            symbol))))

    (t
     (error
      "No se reconoce el símbolo: ~a"
      symbol))))
#+END_SRC


Usa =generate-from-symbol= para intentar construir una cadena completa a partir del símbolo inicial. 
Repite intentos hasta obtener una cadena nueva y válida o hasta agotar un máximo de intentos.

En forma resumida:

1. reinicia contadores temporales;
2. genera una cadena desde el símbolo inicial;
3. descarta la cadena si viola restricciones;
4. la descarta también si ya ha sido generada;
5. si sobrevive a los filtros, la guarda y la devuelve.

#+BEGIN_SRC lisp :tangle ../src/ivns-string-generator.lisp
(defun generate-string
       (&optional
        (start-symbol 'S)
        (max-attempts
         (get-config :gen-max-attempts 500)))

  "Genera una cadena candidata a partir de START-SYMBOL.

   No registra la cadena en GENERATED-STRINGS. El registro se realiza
   después, cuando la cadena ya pasó los filtros."

  (let ((attempts 0)
        (result-string nil)
        (local-index 1))

    (loop while (and (< attempts max-attempts)
                     (null result-string))

          do
            (setf *r-count* 0)

            (handler-case
                (setf result-string
                      (generate-from-symbol
                       start-symbol
                       :productions p-productions
                       :term-list terminals
                       :non-term-list non-terminals
                       :index local-index))

              (r-limit-exceeded ()
                (setf result-string nil))

              (error ()
                (setf result-string nil)))

            (incf attempts))

    result-string))
#+END_SRC

*** =filter-string=

Esta es la función principal que debe llamarse cuando se necesita una
cadena IVNS lista para usar.

La función no acepta cualquier cadena generada por la gramática. Primero
genera una candidata y luego verifica que cumpla los filtros activos.

Su lógica es:

1. generar una cadena candidata con =generate-string=;
2. convertir la lista de símbolos en texto;
3. validar longitud, balance, complejidad y estructura;
4. rechazar la cadena si no cumple;
5. repetir hasta encontrar una cadena válida o agotar los intentos.

Esta función es importante para los experimentos porque evita que IVNS
ejecute vecindades demasiado costosas o estructuralmente inválidas.

=filter-string-with-diagnostics=: genera una cadena IVNS y devuelve información diagnóstica completa sobre el proceso de generación. 
Además de la cadena generada, informa si fue válida, cuántos intentos se hicieron, cuántas cadenas fueron válidas o inválidas 
y cuál fue el motivo del último rechazo.

=filter-string=: mantiene la interfaz simple del generador de cadenas IVNS. 
Internamente usa =filter-string-with-diagnostics=, pero devuelve solamente la cadena válida generada 
o =NIL= si no pudo generarse una cadena aceptable.

#+BEGIN_SRC lisp :tangle ../src/ivns-string-generator.lisp
(defun filter-string-with-diagnostics
       (&key
          (max-attempts
           (get-config :max-attempts 50))

          (max-imbalance
           (get-config :max-imbalance 2))

          (max-length
           (get-config :max-length 10))

          (max-complexity
           (get-config :max-complexity 14))

          (allow-expensive-patterns
           (get-config :allow-expensive-patterns nil)))

  "Genera una cadena IVNS válida y devuelve diagnósticos del filtrado.

   Devuelve siete valores:

   1. cadena válida o NIL;
   2. intentos realizados;
   3. cantidad de cadenas inválidas;
   4. última razón de rechazo;
   5. plist con conteos por razón de rechazo;
   6. cantidad de cadenas generadas;
   7. cantidad de cadenas válidas.

   La cadena solo se registra en GENERATED-STRINGS después de pasar todos
   los filtros. Esto evita que el historial quede contaminado con cadenas
   inválidas."

  (let ((attempts 0)
        (generated-count 0)
        (valid-count 0)
        (invalid-count 0)
        (generated-string nil)
        (last-reason "")
        (reason-counts nil))

    (labels
        ((register-reason
             (reason)

           (let* ((key
                   (ivns-reject-reason->key reason))

                  (current
                   (getf reason-counts key 0)))

             (setf (getf reason-counts key)
                   (1+ current))

             (setf last-reason
                   (format nil "~a" reason)))))

      (loop while (and (< attempts max-attempts)
                       (null generated-string))

            do
              (incf attempts)

              (let* ((generated-list
                      (generate-string
                       'S
                       (get-config :gen-max-attempts 500)))

                     (candidate
                      (when generated-list
                        (join-strings generated-list))))

                (incf generated-count)

                (cond

                  ((null candidate)
                   (incf invalid-count)
                   (register-reason "generation-limit-reached"))

                  ((string-seen-p candidate)
                   (incf invalid-count)
                   (register-reason "duplicated-string"))

                  (t
                   (multiple-value-bind
                         (valid-p reason)

                       (ivns-string-valid-p
                        candidate
                        :max-imbalance max-imbalance
                        :max-length max-length
                        :max-complexity max-complexity
                        :allow-expensive-patterns allow-expensive-patterns)

                     (if valid-p

                         (progn
                           (setf generated-string candidate)
                           (incf valid-count)
                           (add-string candidate))

                         (progn
                           (incf invalid-count)
                           (register-reason reason)))))))))

    (values
     generated-string
     attempts
     invalid-count
     last-reason
     reason-counts
     generated-count
     valid-count)))

        (max-attempts

         (get-config :max-imbalance 2))

         (get-config :max-length 10)))

  "Genera una cadena IVNS que pasa todos los filtros activos.

* Archivo: ivns-sequence-interpreter.lisp

Esta sección procesa la cadena generada y la transforma en una secuencia
de acciones más concreta.

Su responsabilidad es:

- detectar ambigüedad estructural;
- asignar subíndices;
- producir una lista de acciones;
- producir una lista de tokens.

** Idea general de la sección

La salida del generador todavía es demasiado abstracta para usarse directamente en el VRP. 
En esta parte se le da significado operativo a la cadena. Es decir, aquí se decide cómo leer cada símbolo,
cómo numerar rutas, clientes y subrutas, y cómo transformar una secuencia compacta de letras en una lista explícita de acciones entendibles.

** Funciones auxiliares

- =zero-indexes=: devuelve las posiciones de rutas aún no marcadas como usadas.
- =choose-route-index=: escoge una ruta disponible o, si no hay ninguna libre, indica que no fue posible reservar una.
- =pick-one-random=: toma un elemento aleatorio de una lista y lo elimina de ella.
- =join-strings=: une fragmentos de texto en una cadena única.

Estas funciones sirven de apoyo para la interpretación principal. 
Manejan decisiones pequeñas pero frecuentes, como elegir una ruta, sacar un cliente al azar o unir partes de una salida textual.

(defun zero-indexes (routes)
  "Devuelve los índices de las rutas libres (donde el valor es 0)."
  (loop for i from 0 below (length routes)
        when (= (nth i routes) 0)
        collect i))

(defun choose-route-index (routes)
  "Elige aleatoriamente un índice de ruta disponible."
  (let ((available-indexes (zero-indexes routes)))
    (if available-indexes
        (let ((idx (nth (random (length available-indexes)) available-indexes)))
          (values idx T))
        (values nil nil))))

(defun pick-one-random (lst)
  "Elimina y retorna un elemento aleatorio de la lista."
  (if (null lst)
      (values nil lst)
      (let* ((n (length lst))
             (idx (random n))
             (elem (nth idx lst)))
        (values elem (append (subseq lst 0 idx) (subseq lst (1+ idx)))))))

(defun join-strings (parts)
  "Concatena una lista de strings en un solo string."
  (apply #'concatenate 'string parts))

** Validación estructural de cadenas

La gramática puede producir muchas cadenas diferentes, pero no todas son
convenientes para ejecutar sobre una solución CVRP. Algunas pueden ser muy
largas, estar desbalanceadas, generar vecindarios demasiado costosos o no
tener sentido desde el punto de vista de las operaciones que representan.

Por eso, antes de traducir una cadena a criterio VRP, se aplican varios
filtros.

Las funciones de esta sección cumplen los siguientes roles:

- =ivns-complexity-score=:
  calcula una medida simple de complejidad de la cadena.

- =ivns-frequency-balanced-p=:
  verifica que la cantidad de símbolos de selección e inserción no esté
  demasiado desbalanceada.

- =ivns-balanced-structure-p=:
  revisa el balance general entre operaciones que abren selecciones y
  operaciones que las consumen.

- =ivns-expensive-pattern-p=:
  detecta patrones que suelen producir vecindarios caros, repetitivos o poco
  útiles.

- =ivns-semantic-structure-valid-p=:
  valida que la cadena tenga sentido operativo mínimo. Por ejemplo, no
  permite insertar un cliente si antes no fue seleccionado.

- =ivns-string-valid-p=:
  reúne todos los filtros anteriores y devuelve una decisión final sobre la
  cadena.

- =ivns-reject-reason->key=: convierte una razón textual de rechazo de una cadena IVNS en una clave simbólica estable. 
Esta clave permite contar automáticamente los motivos de rechazo en los CSV, como cadenas demasiado largas, 
desbalanceadas, duplicadas o semánticamente inválidas.

Estas funciones son una capa de seguridad entre la generación gramatical y
la ejecución sobre el CVRP. Su objetivo es evitar que IVNS pierda tiempo en
vecindades inválidas, demasiado costosas o imposibles de interpretar.

*** =check-ambiguity=

Esta función conserva el nombre usado en versiones anteriores del proyecto,
pero en la versión actual funciona como una validación estructural.

Su objetivo es detectar si una cadena IVNS puede interpretarse de forma
segura antes de traducirla a acciones.

Por ejemplo, rechaza casos como:

- insertar un cliente sin haber seleccionado uno antes;
- intercambiar dos clientes cuando solo hay uno disponible;
- insertar una subruta sin haber seleccionado una subruta;
- terminar la cadena dejando clientes o subrutas sin consumir.

La función devuelve tres valores:

1. si existe un problema estructural;
2. la cadena analizada;
3. la razón textual de la decisión.

Esta función ayuda a explicar por qué una cadena generada no debe llegar
a la fase de construcción del criterio VRP.

(in-package :ivns)

(defun ivns-complexity-score (cadena)

  "Calcula una medida simple de complejidad para una cadena IVNS."

  (if (or (null cadena)
          (not (stringp cadena)))

      0

      (let ((score 0))

        ;; costo base por longitud
        (incf score
              (length cadena))

        ;; selección de rutas
        (incf score
              (* 2
                 (count #\r cadena)))

        ;; selección e inserción de clientes
        (incf score
              (* 1
                 (count #\a cadena)))

        (incf score
              (* 1
                 (count #\b cadena)))

        ;; intercambio de clientes
        (incf score
              (* 3
                 (count #\c cadena)))

        ;; selección e inserción de subrutas
        (incf score
              (* 2
                 (count #\f cadena)))

        (incf score
              (* 2
                 (count #\g cadena)))

        ;; intercambio de subrutas
        (incf score
              (* 4
                 (count #\e cadena)))

        score)))


(defun ivns-frequency-balanced-p
       (cadena
        &key
          (limite-desequilibrio
           (get-config :max-imbalance 2)))

  "Filtro de balance simple por frecuencia de símbolos."

  (if (or (null cadena)
          (not (stringp cadena)))

      nil

      (let* ((r-count
              (count #\r cadena))

             (a-count
              (count #\a cadena))

             (b-count
              (count #\b cadena))

             (c-count
              (count #\c cadena))

             (f-count
              (count #\f cadena))

             (g-count
              (count #\g cadena))

             (e-count
              (count #\e cadena))

             (client-open
              a-count)

             (client-close
              (+ b-count
                 (* 2 c-count)))

             (subroute-open
              f-count)

             (subroute-close
              (+ g-count
                 (* 2 e-count))))

        (and
         (> r-count 0)

         (<= (abs (- client-open
                     client-close))
             limite-desequilibrio)

         (<= (abs (- subroute-open
                     subroute-close))
             limite-desequilibrio)))))


(defun ivns-balanced-structure-p (cadena)

  "Verifica si la cadena tiene una estructura razonablemente balanceada."

  (if (or (null cadena)
          (not (stringp cadena)))

      nil

      (let* ((opens
              (+ (count #\a cadena)
                 (count #\f cadena)))

             (closes
              (+ (count #\b cadena)
                 (* 2
                    (count #\c cadena))
                 (count #\g cadena)
                 (* 2
                    (count #\e cadena))))

             (max-imbalance
              (get-config :max-imbalance 2)))

        (<= (abs (- opens closes))
            max-imbalance))))


(defun ivns-expensive-pattern-p (cadena)

  "Detecta patrones que suelen generar vecindarios caros o cadenas poco útiles."

  (and cadena

       (or

        ;; repeticiones de rutas / clientes / subrutas
        (search "rfrfrf" cadena)
        (search "rarara" cadena)

        ;; demasiadas selecciones consecutivas
        (search "aaaa" cadena)
        (search "ffff" cadena)

        ;; demasiadas inserciones consecutivas
        (search "bbbb" cadena)
        (search "gggg" cadena)

        ;; swaps repetidos
        (search "ccc" cadena)
        (search "eee" cadena)

        ;; patrones que aparecen frecuentemente como vecinos NIL
        (search "rfff" cadena)
        (search "fffe" cadena)
        (search "fffeg" cadena)
        (search "rfffe" cadena)

        ;; combinaciones largas de clientes
        (search "raaa" cadena)
        (search "aaac" cadena))))


(defun ivns-semantic-structure-valid-p (cadena)

  "Valida la estructura semántica mínima de una cadena IVNS.

   Devuelve tres valores:
   1. T o NIL;
   2. razón textual;
   3. clave estable de rechazo.

   Esta función no evalúa el VRP real. Solo comprueba que la cadena no
   intente usar clientes, rutas o subrutas sin haberlos seleccionado antes."
  (if (null cadena)

      (values nil "Cadena NIL." :unknown)

      (let ((routes 0)
            (clients 0)
            (subroutes 0)
            (valid t)
            (reason ""))

        (loop
          for ch across cadena
          while valid
          do
            (case ch

              (#\r
               (incf routes))

              (#\a
               (if (> routes 0)
                   (incf clients)
                   (progn
                     (setf valid nil)
                     (setf reason "Aparece 'a' sin ruta seleccionada."))))

              (#\b
               (cond
                 ((<= routes 0)
                  (setf valid nil)
                  (setf reason "Aparece 'b' sin ruta disponible."))

                 ((<= clients 0)
                  (setf valid nil)
                  (setf reason "Aparece 'b' sin cliente seleccionado."))

                 (t
                  (decf clients))))

              (#\c
               (if (>= clients 2)
                   (decf clients 2)
                   (progn
                     (setf valid nil)
                     (setf reason "Aparece 'c' sin dos clientes seleccionados."))))

              (#\f
               (if (> routes 0)
                   (incf subroutes)
                   (progn
                     (setf valid nil)
                     (setf reason "Aparece 'f' sin ruta seleccionada."))))

              (#\g
               (cond
                 ((<= routes 0)
                  (setf valid nil)
                  (setf reason "Aparece 'g' sin ruta disponible."))

                 ((<= subroutes 0)
                  (setf valid nil)
                  (setf reason "Aparece 'g' sin subruta seleccionada."))

                 (t
                  (decf subroutes))))

              (#\e
               (if (>= subroutes 2)
                   (decf subroutes 2)
                   (progn
                     (setf valid nil)
                     (setf reason "Aparece 'e' sin dos subrutas seleccionadas."))))

              (otherwise
               (setf valid nil)
               (setf reason
                     (format nil
                             "Símbolo IVNS desconocido: ~a."
                             ch)))))

        (cond
          ((not valid)
           (values nil reason :semantic-invalid))

          ((> clients 0)
           (values nil
                   "La cadena termina con clientes seleccionados sin consumir."
                   :semantic-invalid))

          ((> subroutes 0)
           (values nil
                   "La cadena termina con subrutas seleccionadas sin consumir."
                   :semantic-invalid))

          (t
           (values t "ok" :ok))))))

** Diferencia entre validar e interpretar

En este archivo hay dos pasos distintos:

1. validar la cadena;
2. interpretar la cadena.

Validar significa comprobar que la cadena tiene sentido mínimo. Por
ejemplo, no se debe insertar un cliente si antes no fue seleccionado.

Interpretar significa convertir cada símbolo en una acción concreta del
VRP. Por ejemplo, convertir =a= en “seleccionar cliente C1 desde la ruta
R1”.

Esta separación evita que el traductor reciba cadenas imposibles.

** Función central de interpretación

*** =apply-subindices=

Esta es la función central del intérprete IVNS.

Recibe una cadena como texto y la convierte en una representación más rica,
formada por acciones y tokens.

Por ejemplo, una cadena como =raac= deja de verse como letras aisladas y
pasa a representar una secuencia de acciones sobre rutas y clientes.

La función produce tres salidas:

1. una cadena anotada;
2. una lista legible de acciones;
3. una lista de tokens estructurados.

La lista legible de acciones es útil para depuración y documentación. La
lista de tokens estructurados es útil para validación y traducción
automática.

Esta función es el puente entre el lenguaje formal de IVNS y la construcción
de criterios de vecindad ejecutables.

#+BEGIN_SRC lisp :tangle ../src/ivns-sequence-interpreter.lisp
(defun apply-subindices (input-string)

  "Procesa una cadena IVNS y genera:
   1. cadena anotada;
   2. lista de acciones legibles;
   3. lista de tokens estructurados.

   Antes de interpretar, valida la cadena para evitar que lleguen
   secuencias imposibles al traductor de acciones."

  (multiple-value-bind
        (valid-p reason)

      (ivns-string-valid-p input-string)

    (unless valid-p
      (error "Cadena IVNS rechazada por filtros: ~a | cadena=~a"
             reason
             input-string)))

  (let ((routes nil)
        (clients nil)
        (next-client-id 1)
        (last-picked-client nil)
        (subroutes nil)
        (next-subroute-id 1)
        (last-picked-subroute nil)
        (tokens nil)
        (l-tokens nil)
        (list-string nil)
        (any-annotation? t))

    (loop for ch across input-string do

          (case ch

            (#\r
             (setf routes
                   (append routes
                           (list 0)))

             (let ((token
                    (format nil
                            "r_~d"
                            (length routes))))

               (push token tokens))

             (push

* Traducción de acciones IVNS a criterio VRP

Esta sección conecta el lenguaje formal de IVNS con el framework VRP.

Hasta este punto, el sistema ya puede:

1. generar una cadena;
2. validarla;
3. interpretarla como acciones.

Pero el framework VRP no ejecuta directamente cadenas de texto. Necesita
un criterio de vecindad en su formato interno.

Por eso esta sección traduce acciones IVNS a operaciones VRP.

El archivo generado es:

- =../src/vrp-ivns.lisp=

El flujo de traducción es:

1. convertir nombres textuales en símbolos del paquete =vrp=;
2. traducir cada acción IVNS a una operación VRP;
3. agrupar las operaciones;
4. construir un criterio ejecutable con =make-neighborhood-criterion=.

Esta sección es el puente entre la parte formal del método y la ejecución
real sobre soluciones CVRP.

#+BEGIN_SRC lisp :tangle ../src/vrp-ivns.lisp

(in-package :vrp)

(defun ivns-route-sym (txt)
  (intern (string-upcase txt) (find-package :vrp)))

(defun ivns-sym (txt)
  "Convierte texto como r1/c2/f1 en símbolo del paquete VRP."
  (intern (string-upcase txt) :vrp))

(defun ivns-client-sym (txt)
  ;; a1, b1, c1  -> C1
  (let ((n (subseq txt 1)))
    (intern (format nil "C~A" (string-upcase n))
            (find-package :vrp))))

(defun ivns-subroute-sym (txt)
  ;; f1, g1, e1 -> Z1
  (let ((n (subseq txt 1)))
    (intern (format nil "Z~A" (string-upcase n))
            (find-package :vrp))))
#+END_SRC


** Funciones principales

*** =ivns-action->vrp-op=

Traduce una sola acción IVNS a su operación equivalente en el lenguaje del VRP. Es un traductor uno-a-uno entre dos representaciones:

- una representación más legible y simbólica del IVNS;
- una representación compatible con el framework VRP.

Por ejemplo, convierte acciones como =select-route= o =swap-clients= en la sintaxis exacta que entiende el motor del VRP.

#+BEGIN_SRC lisp :tangle ../src/vrp-ivns.lisp
(defun ivns-action->vrp-op (action)
  (cond
    ((equal (first action) "select-route")
     `(select-route ,(ivns-route-sym (second action))))

    ((equal (first action) "select-client")
     `(select-client
       ,(ivns-client-sym (second action))
       from
       ,(ivns-route-sym (fourth action))))

    ((equal (first action) "insert-client")
     `(insert-client
       ,(ivns-client-sym (second action))
       into
       ,(ivns-route-sym (fourth action))))

    ((equal (first action) "swap-clients")
     `(swap-clients
       ,(ivns-client-sym (second action))
       ,(ivns-client-sym (third action))))

    ((equal (first action) "select-subroute")
     `(select-subroute
       ,(ivns-subroute-sym (second action))
       from
       ,(ivns-route-sym (fourth action))))

    ((equal (first action) "insert-subroute")
     `(insert-subroute
       ,(ivns-subroute-sym (second action))
       into
       ,(ivns-route-sym (fourth action))))

    ((equal (first action) "swap-subroutes")
     `(swap-subroutes
       ,(ivns-subroute-sym (second action))
       ,(ivns-subroute-sym (third action))))

    (t
     (error "Operacion desconocida: ~a" action))))
#+END_SRC


*** =ivns-actions->criterion=

Recibe una lista completa de acciones IVNS ya interpretadas y construye con ellas un criterio de vecindad utilizable por el VRP.

Su papel es:

1. traducir cada acción individual;
2. agrupar la secuencia traducida;
3. construir el objeto criterio que luego podrá ejecutarse sobre una solución.

Aquí ocurre el puente entre el lenguaje formal del IVNS y el espacio operativo del VRP.
#+BEGIN_SRC lisp :tangle ../src/vrp-ivns.lisp
(defun ivns-actions->criterion
       (actions
        &key
          (print-description nil))

  "Convierte una lista de acciones IVNS en un criterio ejecutable del VRP.

   ACTIONS viene de `apply-subindices`.
   PRINT-DESCRIPTION permite imprimir la descripción generada solo cuando
   se está depurando manualmente. En experimentos debe mantenerse en NIL."

  (let* ((description
          (mapcar #'ivns-action->vrp-op
                  actions))

         (*package*
          (find-package :vrp)))

    (when print-description
      (format t "~%Description generado:~%")
      (dolist (d description)
        (format t "~s~%" d)))

    (eval
     `(make-neighborhood-criterion
       ',description
       +exhaustive-search-strategy+
       +best-improvement+))))
#+END_SRC


** Utilidades de tiempo, gap y evaluación de soluciones

Estas funciones auxiliares se usan durante la ejecución de IVNS para medir
tiempo, calcular calidad relativa y evaluar soluciones.

No forman parte de la gramática ni de la traducción de cadenas. Su papel es
apoyar la ejecución experimental del algoritmo.

Las funciones incluidas son:

- =ivns-current-time=:
  obtiene el tiempo actual en segundos.
- =ivns-elapsed-time=:
  calcula el tiempo transcurrido desde un instante inicial.
- =ivns-compute-gap=:
  calcula el gap porcentual respecto a la mejor solución conocida.
- =ivns-evaluate-solution=:
  evalúa una solución CVRP usando el motor del framework VRP y actualiza su
  costo.

Estas utilidades permiten que IVNS registre métricas comparables con VNS,
especialmente tiempo de ejecución, costo total, penalización y gap respecto
al BKS.

#+BEGIN_SRC lisp :tangle ../src/vrp-ivns.lisp  
(defun ivns-current-time ()
  (/ (get-internal-real-time)
     internal-time-units-per-second))


(defun ivns-elapsed-time (start)
  (- (ivns-current-time) start))


(defun ivns-compute-gap (best known-best)
  (if (and known-best
           (> known-best 0))
      (* 100.0
         (/ (- best known-best)
            known-best))
      0))


(defun ivns-evaluate-solution
    (solution
     problem
     penalty-factor)

  (let ((action
         (basic-cvrp-action
          :penalty-factor penalty-factor)))

    (simulate-solution
     solution
     problem
     action)

    (let ((total
           (+ (total-distance action)
              (total-penalty action))))

      (setf (cost solution)
            total)

      (values
       total
       (total-distance action)
       (total-penalty action)))))
#+END_SRC

** =ivns-run=

Esta función ejecuta una versión básica de IVNS sobre una solución real del
CVRP.

Su objetivo principal es comprobar que el flujo completo funciona:

1. generar una cadena IVNS válida;
2. interpretarla como acciones;
3. convertir las acciones en un criterio VRP;
4. construir una solución inicial;
5. aplicar el criterio sobre la solución;
6. evaluar el vecino generado;
7. aceptar la mejora si reduce el costo;
8. registrar una traza simple de la ejecución.

Esta función es útil para pruebas manuales y depuración.

Para la comparación experimental formal entre VNS e IVNS debe usarse
=ivns-run-experimental=, definido en =experiments-vns-ivns.org= y tanglado
en =../src/ivns-experiments.lisp=.


#+BEGIN_SRC lisp :tangle ../src/vrp-ivns.lisp       
;; ============================================
;; EJECUCIÓN REAL
;; ============================================
(defun ivns-run
    (&key
       (problem a-n33-k5-problem)
       (max-iter 100)
       (penalty-factor 1000)
       (known-best nil)
       (quiet nil))

  (format t "~%======================================")
  (format t "~% IVNS + VRP")
  (format t "~%======================================~%")

  (let* ((start-time
          (ivns-current-time))

         ;; solucion actual
         (current-solution
          (make-initial-solution-for-cvrp-random
           problem))

         ;; mejor global
         (best-global nil)

         ;; acciones auxiliares
         (delta-act nil)

         ;; costos
         (initial-cost 0)
         (current-cost 0)
         (best-cost 0)

         ;; control
         (accepted-improvements 0)
         (rejected-moves 0)
         (invalid-neighbors 0)

         ;; trazas
         (used-strings '())
         (trace '()))

    ;; ===================================
    ;; COSTO INICIAL
    ;; ===================================

    (multiple-value-bind
          (cost distance penalty)

        (ivns-evaluate-solution
         current-solution
         problem
         penalty-factor)

      (declare (ignore distance penalty))

      (setf initial-cost cost)
      (setf current-cost cost)
      (setf best-cost cost)
      (setf best-global
            (clone current-solution)))

    ;; ===================================
    ;; MOSTRAR INICIO
    ;; ===================================

    (unless quiet
      (format t "~%[SOLUCION INICIAL]")
      (format t "~%Costo: ~a~%" current-cost)
      (pp-solution current-solution t))

    ;; ===================================
    ;; LOOP PRINCIPAL
    ;; ===================================

    (loop for iter from 1 to max-iter do

      (unless quiet
        (format t "~%===================================")
        (format t "~%ITERACION ~a" iter)
        (format t "~%===================================~%"))

      (let* ((iter-start
              (ivns-current-time))

             (cadena
              (ivns::filter-string))

             (accepted nil)

             (delta 0)

             (neighbor nil)

             (distance 0)

             (penalty 0))

        (push cadena used-strings)

        (unless quiet
          (format t "~%Cadena generada: ~a~%"
                  cadena))

        (multiple-value-bind
              (res actions tokens)

            (ivns::apply-subindices cadena)

          (declare (ignore res tokens))

          (let ((criterion
                 (ivns-actions->criterion
                  actions)))

            ;; accion delta
            (setf delta-act
                  (delta-cvrp-action))

            ;; ejecutar criterio
            (setf neighbor
                  (funcall criterion
                           current-solution
                           problem
                           delta-act))

            ;; ============================
            ;; SI HAY VECINO
            ;; ============================

            (if neighbor

                (multiple-value-bind
                      (new-cost dist pen)

                    (ivns-evaluate-solution
                     neighbor
                     problem
                     penalty-factor)

                  (setf distance dist)
                  (setf penalty pen)

                  (setf delta
                        (- new-cost
                           current-cost))

                  ;; mejora
                  (if (< delta 0)

                      (progn

                        (setf accepted t)

                        (incf accepted-improvements)

                        (setf current-solution
                              neighbor)

                        (setf current-cost
                              new-cost)

                        ;; mejor global
                        (when (< current-cost
                                 best-cost)

                          (setf best-cost
                                current-cost)

                          (setf best-global
                                (clone neighbor)))

                        (unless quiet
                          (format t
                                  "~%[ITER ~3d] MEJORA | Costo: ~6d | Cadena: ~a"
                                  iter
                                  current-cost
                                  cadena)

                          (when (= current-cost
                                   best-cost)

                            (format t
                                    "~%          >>> NUEVO GLOBAL: ~a"
                                    best-cost))

                          (format t "~%")

                          (pp-solution
                           current-solution
                           t)))

                      ;; no mejora
                      (progn

                        (incf rejected-moves)

                        (unless quiet
                          (format t
                                  "~%[ITER ~3d] SIN MEJORA | Actual: ~6d | Cadena: ~a"
                                  iter
                                  current-cost
                                  cadena)))))

                ;; vecino nil
                (progn

                  (incf invalid-neighbors)

                  (unless quiet
                    (format t
                            "~%[ITER ~3d] VECINO NIL | Cadena: ~a"
                            iter
                            cadena)))))

          ;; ===================================
          ;; GUARDAR TRACE
          ;; ===================================

          (push
           (list
            :iter iter
            :string cadena
            :accepted accepted
            :current-cost current-cost
            :best-cost best-cost
            :delta delta
            :distance distance
            :penalty penalty
            :time
            (ivns-elapsed-time iter-start))
           trace))))

    ;; ===================================
    ;; FINAL
    ;; ===================================

    (let* ((total-time
            (ivns-elapsed-time start-time))

           (gap
            (ivns-compute-gap
             best-cost
             known-best)))

      (format t "~%======================================")
      (format t "~% RESULTADO FINAL")
      (format t "~%======================================")

      (format t "~%Initial cost: ~a"
              initial-cost)

      (format t "~%Best cost: ~a"
              best-cost)

      (format t "~%Final cost: ~a"
              current-cost)

      (format t "~%Iterations: ~a"
              max-iter)

      (format t "~%Accepted improvements: ~a"
              accepted-improvements)

      (format t "~%Rejected moves: ~a"
              rejected-moves)

      (format t "~%Invalid neighbors: ~a"
              invalid-neighbors)

      (format t "~%Unique strings: ~a"
              (length
               (remove-duplicates
                used-strings
                :test #'equal)))

      (format t "~%Execution time: ~,4f sec"
              total-time)

      (format t "~%Gap: ~,4f~%"
              gap)

      (format t "~%======================================~%")

      ;; ===================================
      ;; RETORNO EXPERIMENTAL
      ;; ===================================

      (list
       :algorithm :ivns

       :initial-cost initial-cost
       :best-cost best-cost
       :final-cost current-cost

       :iterations max-iter

       :accepted-improvements
       accepted-improvements

       :rejected-moves
       rejected-moves

       :invalid-neighbors
       invalid-neighbors

       :execution-time total-time

       :gap gap

       :trace
       (reverse trace)

       :strings-used
       (reverse used-strings)

       :best-solution
       best-global))))

import csv

rows = list(csv.DictReader(summary.open(newline="", encoding="utf-8"))) if summary.exists() else []

print("trace rows:", len(trace_rows))

def num(r, c):
    try:
        return float(r.get(c, 0) or 0)
    except Exception:
        return 0.0

problems = []

for r in rows:
    if r.get("algorithm") != "ivns":
        problems.append(("algorithm inesperado", r.get("algorithm")))

        "ivns_strings_generated_total",

        "ivns_strings_evaluated_total",
        "ivns_strings_selected_total",
    ]:
        if num(r, c) <= 0:
            problems.append((r.get("run_id"), c, r.get(c)))

print("problemas:", problems if problems else "OK")
