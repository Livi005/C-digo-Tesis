        "This function returns the code to be inserted in the do-neighborhood-operation macros to process all simpler-ops than can be obtained from a given neighborhood operation."

           ;; here we process all the operations

             ;; push the current op into ops-list

             ;; simulate the operation

              ,simple-op ,working-copy))))

**** make-compute-route-end-code
***** description
      When writing the code for a do-neighborhood-operation macro that traverse a route (such as doselect-client or doinsert-client) it is necessary to compute how far into route we should repeat the operation.

      For instance, in the case of the select-client, we should traverse the route up to the last position, because we can select all the clients in the route including the last one.  In the case of an client-insertion, we should traverse the list up to one position more than the route length because we can insert at the end of the route.  If we are dealing with a select-subroute, then we can select from the subroute up to the position =route-length - subroute-length + 1=, because we need to select the subroutes with a fixed length.

      This function returns the code to compute the last position to do the operation according to the operation we are dealing with.  On a first approach I'm going to make it a generic function.
***** generic-function
     #+BEGIN_SRC lisp +n -r :results none :exports code :tangle "../src/vrp-delta-cost.lisp"
     (defgeneric make-compute-route-end-code (operation working-copy)
       (:documentation "Returns an expression to compute the last position in a route to repeat the operation `operation' in a do-neighborhood-operation macro."))
     #+END_SRC

      (defmethod make-compute-route-end-code
          ((op operation-select-client) (wc basic-working-copy))
        "Returns an expression to compute the last position in a route to repeat the operation `operation' in a do-neighborhood-operation macro. If the operation is a select-client, then we need to iterate up to the last position in the route."

**** make-op-instance-code
***** description
      When writing the code for a do-neighborhood-operation macro one of the most important parts is the creation of the operation's instance.

      - In the case of the select-client, before we create the class, we need to bind cname to =(1+ (selections-count ,working-copy))= and then use that value as an argument te op-select-client.

      - In the case of an insert-client operation we don't bind =cname= and in the constructor of the operation we just use the value that the user passed to us in =cname=.

      - In the case of a select-subroute, we need to bind =cname= to =(1+ (selections-count ,working-copy))=, and we need to pass to the constructor the size of the subroute.

      The idea is that the function that we are going to create in this section will deal with this aspects, and will return the code to insert in the loop in the corresponding do-neighborhood-operation macro.

** make-compute-route-end-code
*** description
    When writing the code for a do-neighborhood-operation macro that traverse a route (such as doselect-client or doinsert-client) it is necessary to compute how far into route we should repeat the operation.

    For instance, in the case of the select-client, we should traverse the route up to the last position, because we can select all the clients in the route including the last one.  In the case of an client-insertion, we should traverse the list up to one position more than the route length because we can insert at the end of the route.  If we are dealing with a select-subroute, then we can select from the subroute up to the position =route-length - subroute-length + 1=, because we need to select the subroutes with a fixed length.

    This function returns the code to compute the last position to do the operation according to the operation we are dealing with.  On this approach I'm going to make it a macro.
